# Anti-Slop

## 目标

这份文件不是“审美偏好清单”，而是拦住通用 AI 页面味的实现前自检。
每次界面任务都要先有具体审美主张，再用字体、色彩、动效、构图和素材把它执行出来。

## 禁区：通用 AI 味

- 三版候选都只是浅色留白加点缀色
- 内容型页面没有真实图，只用大面积占位块
- 工具型页面做成营销站
- 页面里出现多个互相冲突的圆角、阴影和边框体系
- 不核实事实就写产品页
- 默认紫色/蓝紫渐变、玻璃卡、光晕背景、白底圆角卡片阵列，且没有品牌或参考图依据
- 默认使用 Inter、Roboto、Arial 或 system-ui 作为审美选择；除非它服务于系统一致性、可访问性或高密度扫描
- 三个方向共享同一个 hero、同一组卡片、同一个视觉重心，只换颜色、插图或按钮文案
- 落地页总是“左侧大标题 + 右侧插图/截图 + 下方三卡片”
- 用纹理、噪点、渐变、阴影或动效掩盖事实、素材、信息层级或任务路径缺失
- 页面只有一个颜色家族的深浅变化，没有清晰的主次色、功能色和情绪色分工
- 极简页面缺少精确排版、比例、边距和文本节奏；复杂页面缺少秩序、层级和可扫描路径
- 所有页面都像同一个安全模板，没有与用户、场景、内容和品牌相关的记忆点

## 允许例外

- 用户、品牌规范、参考图或现有设计系统明确要求某种颜色、字体或布局时，以这些约束为准。
- 工具型、企业型、合规型界面可以保持克制，但必须通过密度、对齐、状态、表格、控件和信息结构体现设计质量。
- 中性字体可以用于正文和高密度 UI，但标题、数字、标签或关键模块仍应有明确角色分工。

## 优先替代

- 用明确 lens 代替“做个好看的页面”
- 用 theme lock 代替边写边试色
- 用 layout skeleton 代替从空白页硬搭
- 用真实内容资产代替装饰性占位
- 用“用途 / 受众 / 气质 / 约束 / 记忆点”代替“现代、简洁、高级”
- 用有角色分工的标题字体、正文字体、数字字体和标签样式代替单一通用字体栈
- 用具体色彩角色代替默认紫色渐变：主动作色、情绪强调色、功能信息色、弱化背景色要各司其职
- 用一段有节制的入场、滚动或状态动效代替到处散落的 hover 小把戏
- 用不对称、密度变化、图文节奏、真实截图/图片和局部重点放大代替均匀卡片堆叠
- 用 `visual-compare` 检查实现是否保住气质、层级和记忆点，而不只是检查有没有摆对位置
