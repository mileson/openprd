# 三方向引擎

## 目标

让 3 个候选方向真的来自不同思路，而不是安全解的三个轻微变体。
每个方向都必须先有审美主张，再有页面骨架；不要先套模板，再给模板找理由。

## 前提

- 如果用户已经给了确认效果图、设计稿或明确参考图，先不要强行再出 3 个方向；先吸收它的版式、层级、主视觉和关键路径。
- 这时方向引擎的任务变成：找最接近该参考图的 starter / theme / layout 作为实现加速器；如果库里没有接近项，也允许偏离默认组合，以参考图为准。

## 审美立意输入

每个方向先回答 5 个问题：

- 用途：这个界面解决什么问题，用户为什么打开它。
- 受众：用户是来快速完成工作、被说服、浏览内容，还是沉浸探索。
- 气质：选择一个明确端点，例如克制极简、工业工具感、编辑杂志感、复古未来、自然有机、精致奢华、玩具感、艺术几何、温暖邀请、密集控制台。
- 约束：框架、性能、可访问性、真实素材、品牌资产、信息密度和移动端限制。
- 记忆点：用户看完后应该记住的一件视觉或交互特征。

## 反模板化约束

- 不要默认使用紫白渐变、蓝紫光晕、玻璃卡片、白底圆角卡片堆叠、通用 SaaS hero 或“左文案右插图”的安全布局。
- 不要默认把 Inter、Roboto、Arial 或 system-ui 当成审美选择；除非品牌或系统限制要求，否则要用更有角色分工的标题/正文字体组合。
- 不要让 3 个方向共享同一个视觉重心、同一组卡片、同一套浅色留白，只换色或换图。
- 不要用装饰性纹理、阴影、动效、渐变来掩盖事实、素材或信息层级缺失。
- 如果必须采用克制中性方案，说明它服务的是扫描效率、可访问性或用户工作流，而不是“先做个安全版”。

## 方向类型

### 1. Contrast

- 主动拉开明暗、密度、节奏、叙事方式、视觉温度或动效强度。
- 适合用户没有明确风格、但需要先看到差异时。

### 2. Reference Transfer

- 从真实世界成熟范式中迁移结构方法。
- 借的是组织方式、信息优先级和节奏，不是照着抄外观。
- 参考来源可以是杂志目录、展陈导览、控制台、奢侈品目录、工具面板、游戏 HUD、报纸版式或专业仪表盘，但必须抽象成 OpenPRD 自己的设计语言。

### 3. Design Lens

- 先选一个明确的设计哲学，再从这个 lens 反推页面结构。
- 例如工具密度优先、展陈叙事优先、产品发布优先、编辑排版优先。

## 每个方向至少回答

- 这个方向的视觉主张是什么
- 它的气质端点是什么
- 用户会记住什么
- 它为什么和另外两版不一样
- 它更适合什么任务和用户
- 它最大的风险是什么
- 它主动避开了哪种通用 AI 模板味

## 不合格示例

- 三版都只是浅色留白加不同插图
- 三版都只是按钮颜色不同
- 三版都共享同一布局和同一视觉重心，只是文案换序
- 三版都用紫色渐变、玻璃卡、白底卡片阵列或同一个通用字体栈
- 方向说明只写“现代、简洁、高级”，没有具体气质和记忆点

## 合格示例

- 一版强调沉浸式主视觉和少量强信号，记忆点是首屏里的真实产品或场景状态。
- 一版强调编辑感和内容节奏，记忆点是标题字体、图文节拍和章节式浏览。
- 一版强调密度、扫描效率和操作导向，记忆点是稳定控制台结构、清晰状态和可反复使用的工具感。
